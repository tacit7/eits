Agent Cron Scheduling — Pick an approach

How should users create scheduled agent runs? Pick the one that fits your mental model best.

A

Enhance existing Job form

When job type = "Spawn Agent", replace the free-text instructions field with an agent template picker (dropdown of saved prompts). Still lives inside the current job drawer. Minimal change, maximum reuse.

Pros

  • No new pages
  • Works in both global + project job views
  • Consistent with how other job types work

Cons

  • Drawer is already getting tall
  • Agent-specific context is buried in a generic form
B

Dedicated "Agent Scheduler" tab/page

A first-class page (or tab inside Jobs) specifically for scheduling agents. Shows agent cards with a schedule button. Scheduling opens a stripped-down form: just schedule + model, with the agent/project already locked in.

Pros

  • Clean agent-centric UX
  • Surfaced as its own feature, not buried in job types

Cons

  • New page/route needed
  • Duplicates some job management UI
C

"Schedule this agent" from Prompts page

Add a Schedule button directly on each prompt card in the existing Prompts UI. Clicking it pre-fills a job creation form with that agent's instructions and project, so the user just picks a schedule and saves.

Pros

  • Contextual — schedule from where you manage agents
  • Least new code

Cons

  • Discoverability — user has to go to Prompts to create a job
  • Doesn't help from the Jobs page
