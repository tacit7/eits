# EITS Web Code Audit — 2026-03-10

Scope: Full read-only audit of the Phoenix/Elixir codebase for code quality,
readability, best practices, and tech debt.

---

## CRITICAL

### 1. DateTime Fields Stored as Strings
**File:** `lib/eye_in_the_sky_web/sessions/session.ex:19-28`

`started_at`, `ended_at`, `last_activity_at`, `archived_at` are all declared
as `:string` fields. The codebase calls `DateTime.utc_now() |>
DateTime.to_iso8601()` at 20+ call sites to produce strings for storage.

Problems:
- Loses all native DB date functions (range queries, `DATE_TRUNC`, index use)
- Comparisons are lexicographic; only works because ISO 8601 sorts correctly
- `archive_session/1` (sessions.ex:144) uses `DateTime.to_string()` instead of
  `to_iso8601()`. These produce different formats (to_string includes timezone
  abbreviation). Silent inconsistency in the same column.

Fix: Migrate fields to `:utc_datetime`. Use `timestamps()` in schemas.

---

### 2. Context Boundary Violation in DmLive
**File:** `lib/eye_in_the_sky_web_web/live/dm_live.ex:4, 147-157`

DmLive imports `alias EyeInTheSkyWeb.Repo` and calls `Repo.insert_all/2`
directly inside the LiveView:

```elixir
Repo.insert_all("task_sessions", [%{task_id: task.id, session_id: session_id}], ...)
Repo.insert_all("task_tags", [%{task_id: task.id, tag_id: tag.id}], ...)
```

`Tasks.link_session_to_task/2` and `Tasks.link_tag_to_task/2` already exist for
exactly this purpose. The LiveView bypasses them, skipping PubSub broadcasts,
validation, and any future hooks.

## HIGH

### 3. AppleScript String Injection Risk
**File:** `lib/eye_in_the_sky_web_web/live/dm_live.ex:316-327`

`dir` (from `resolve_project_path`) is interpolated directly into an AppleScript
heredoc without escaping:

```elixir
write text "cd #{dir} && claude --dangerously-skip-permissions -r #{session_uuid}"
```

Project paths with spaces, quotes, or AppleScript metacharacters will break
the script. No sanitization is applied.

---

### 4. Blocking System.cmd in LiveView Process
**File:** `lib/eye_in_the_sky_web_web/live/dm_live.ex:280-287`

`System.cmd("git", ["-C", project_path, "show", hash, "--unified=5"])` is a
synchronous blocking call in the LiveView process. A slow git invocation blocks
all user event processing until it returns.

Fix: Use `Task.async` and return the result via `handle_info`.

---

### 5. Dead Code — update_agent_status/2 Is a No-op
**File:** `lib/eye_in_the_sky_web/agents.ex:133-136`

```elixir
def update_agent_status(%Agent{} = agent, _status) do
  {:ok, agent}
end
```

Silently ignores the status param and returns the unchanged agent. Callers
believe they updated status; nothing changed. Delete it.

---

### 6. Misleading Function Name — list_active_agents/0
**File:** `lib/eye_in_the_sky_web/agents.ex:38-44`

Returns ALL agents with no status filter. Identical in behavior to `list_agents/0`
plus project preloading. The name implies filtering that does not happen.

---

### 7. Dual Atom/String Key Access Noise in SessionWorker
**File:** `lib/eye_in_the_sky_web/claude/session_worker.ex:360, 419-543`

`Map.get(parsed, "type") || Map.get(parsed, :type)` appears ~25 times.
`Jason.decode/1` always returns string keys. The atom fallback is dead code.

Fix: Commit to string keys and remove all `|| Map.get(parsed, :atom)` fallbacks,
or use `Jason.decode(line, keys: :atoms)` once at the entry point.

## MEDIUM

### 8. Hardcoded Magic State IDs
**Files:** `lib/eye_in_the_sky_web/tasks.ex:54`, `lib/eye_in_the_sky_web/nats/handler.ex:336`

```elixir
# tasks.ex
where([t, ts], ts.session_id == ^session_id and t.state_id == 2)

# nats/handler.ex
state_id = if cmd == "start", do: 2, else: 3
```

Workflow state IDs are hardcoded as magic integers. `Tasks.get_workflow_state_by_name/1`
already exists. If IDs change or rows are reseeded, these break silently.

---

### 9. FTS5 Module Name Is a Legacy Lie
**File:** `lib/eye_in_the_sky_web/search/fts5.ex`

Module `EyeInTheSkyWeb.Search.FTS5` implements PostgreSQL `tsvector/tsquery`,
not SQLite FTS5. The file and module name are leftover from pre-migration.

Fix: Rename to `EyeInTheSkyWeb.Search.PGSearch` with matching file rename.

---

### 10. Raw SQL in get_session_counts/1
**File:** `lib/eye_in_the_sky_web/sessions.ex:370-386`

Uses `Repo.query/2` with a raw 5-subquery SQL string. Bypasses Ecto type safety,
parameterization guarantees, and query composition.

Fix: Replace with parallel `Repo.aggregate/3` calls or a single Ecto query
using `select` with multiple `fragment` subqueries.

---

### 11. Raw SQL LIMIT Interpolation in FTS5
**File:** `lib/eye_in_the_sky_web/search/fts5.ex:74`

```elixir
LIMIT #{effective_limit}
```

Interpolated directly into the SQL string rather than parameterized. Not
user-controlled now, but sets a bad precedent. Use `LIMIT $N`.

---

### 12. require Logger Inside Function Body
**File:** `lib/eye_in_the_sky_web/sessions.ex:152`

`require Logger` is called inside `unarchive_session/1` instead of at module top.
Works but is non-idiomatic.

---

### 13. Aliases Inside Function Bodies (MCP Tools)
**File:** `lib/eye_in_the_sky_web/mcp/tools/todo.ex` and other tool files

Each `execute/2` clause re-declares `alias EyeInTheSkyWeb.Tasks` (or other
contexts) inside the function body. This repeats across 28+ clauses. Aliases
are module-level declarations; move them to module top.

---

### 14. Inconsistent Error Return in find_unlinked_message/3
**File:** `lib/eye_in_the_sky_web/messages.ex:443`

Returns `:not_found` (not `{:error, :not_found}`). Every other lookup function
in the codebase returns `{:error, :not_found}`.

---

### 15. end_session/2 Opts Type Inconsistency
**File:** `lib/eye_in_the_sky_web/sessions.ex:131`

Defined with `opts \\ %{}` (map default), accessed with `opts[:summary]`
(keyword list syntax). Maps support `[:key]` access and return nil, so no
crash — but callers passing `[summary: "..."]` as a keyword list will silently
get nil because the arg was declared as a map. Standardize to `opts \\ []`.

---

### 16. for_session_join Has Hardcoded Join Column
**File:** `lib/eye_in_the_sky_web/query_helpers.ex:57`

```elixir
join(:inner, [x], j in ^join_table, on: j.task_id == x.id)
```

The join column is hardcoded as `task_id`. The helper is only usable for
`task_sessions`. Any other join table with a differently named column will fail.
The generic API is misleading.

---

### 17. try/rescue for Expected Not-Found (Control Flow via Exception)
**Files:** `lib/eye_in_the_sky_web/nats/handler.ex:338-354`, `lib/eye_in_the_sky_web/mcp/tools/todo.ex:163-181, 190-200`

```elixir
try do
  task = Tasks.get_task!(task_id)
  ...
rescue
  Ecto.NoResultsError -> Logger.warning(...)
end
```

Using exception handling for expected not-found flow is an anti-pattern in
Elixir. Add `Tasks.get_task/1` (non-raising) and pattern match on
`{:error, :not_found}`.

---

### 18. PubSub Topic Message Shapes Are Undocumented and Inconsistent
`"agent:working"` receives `{:agent_working, session_id, session_int_id}` from
SessionWorker but `{:agent_working, session_struct}` from NATS.Handler.
`"agents"` broadcasts mix Agent and Session structs under the same event name.
No type specs or module docs define what each topic emits.

---

### 19. NaiveDateTime vs DateTime in Notifications
**File:** `lib/eye_in_the_sky_web/notifications.ex:93`

```elixir
cutoff = NaiveDateTime.utc_now() |> NaiveDateTime.add(-days * 86400, :second)
```

Rest of codebase uses `DateTime.utc_now()`. Mixing naive and aware datetimes
can cause comparison failures depending on column type.

### 20. Unfixed TODO — Notes Always Returns []
**File:** `lib/eye_in_the_sky_web/sessions.ex:358-359`

```elixir
# TODO: Fix parent_id type mismatch (INTEGER vs TEXT)
notes: [],
```

`load_session_data/1` always returns an empty notes list. Broken since the
SQLite-to-PostgreSQL migration. Any caller relying on this for notes silently

### 21. DmLive Too Large — Too Many Concerns
**File:** `lib/eye_in_the_sky_web_web/live/dm_live.ex`

8+ `handle_event` clauses and 6+ `handle_info` clauses covering: file uploads,
model selection, task creation with tag parsing, tab switching with lazy loads,
live streaming, session file reload, diff fetching, and iTerm2 integration.

Streaming, task creation, and file upload logic should be in dedicated
components.

## LOW

### 22. broadcast_change/1 in Tasks Discards Its Argument
**File:** `lib/eye_in_the_sky_web/tasks.ex:297-299`

`defp broadcast_change(_)` — callers pass `{:ok, task}` which is silently
ignored. Subscribers receive only `:tasks_changed` and must re-fetch everything.
At minimum rename param to `_result` to signal intent.

---

### 23. find_or_create_agent Uses rescue for Race Instead of on_conflict
**File:** `lib/eye_in_the_sky_web/nats/handler.ex:368-394`

TOCTOU race is handled with `rescue Ecto.ConstraintError` plus a second lookup.
Idiomatic fix: `Repo.insert/2` with `on_conflict: :nothing` and a unique index,
removing the need for rescue entirely.

---

### 24. @moduledoc After Module Attributes in SessionWorker
**File:** `lib/eye_in_the_sky_web/claude/session_worker.ex:12`

`@moduledoc` appears after `@registry` and `@max_queue_size`. Convention is
`@moduledoc` immediately after `use`/`require`/`import` declarations.

---

### 25. Notes Context Does Extra DB Lookup Per Query
**File:** `lib/eye_in_the_sky_web/notes.ex:24-44`

`list_notes_for_session/1` fetches the Session record first just to get both
integer ID and UUID for dual-ID matching on `parent_id`. Symptom of `parent_id`
being stored as untyped text with no consistent format across the migration.

---

### 26. list_recent_messages Overload Confusion
**File:** `lib/eye_in_the_sky_web/messages.ex:322-383`

Three versions of the function: arity-2 (DB version), arity-3 (JSONL version),
arity-2 again (delegates to arity-3 with nil). Two arity-2 clauses for the same
function is disorienting and fragile.

---

### 27. Emoji-Heavy Logging Hurts grep/Filtering
Emoji prefixes (`🔧 ✅ ❌ 📦 📝 📤 🔄`) throughout session_worker.ex,
agent_manager.ex, sessions.ex, and others. Makes standard log filtering
(`grep ERROR`, `grep WARNING`, logfmt parsers) unreliable.

---

### 28. Hardcoded Git Workflow Instructions in AgentManager
**File:** `lib/eye_in_the_sky_web/claude/agent_manager.ex:80-91`

Git workflow instructions with specific remote names (`gitea`) and CLI tool
names (`tea pr create`) are embedded as a heredoc in application source.
The `subagent_prompts` table already exists for exactly this purpose.
