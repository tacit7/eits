### Which-key overlay (new)

Timed prefix hints for `g`, `t`, `n`:
- **300ms** after prefix key → overlay appears showing available second keys
- **1000ms** after prefix key → sequence expires, buffer clears, overlay hides
- Any keypress dismisses instantly

## Task 2: Which-key overlay

**File:** `assets/js/hooks/vim_nav.ts`

Shows available completions 300ms after a prefix key. Uses the same `COMMANDS` registry as the `?` help overlay.

### New state

```typescript
whichKeyEl: null as HTMLElement | null,
whichKeyTimer: null as ReturnType<typeof setTimeout> | null,
_onWhichKeyClose: null as ((e: KeyboardEvent) => void) | null,
```

### `showWhichKey(prefix: string)`

Schedule `setTimeout(() => this._renderWhichKey(prefix), 300)`, store in `whichKeyTimer`.

### `_renderWhichKey(prefix: string)`

- Filter `COMMANDS` where `cmd.keys[0] === prefix`
- Build fixed bottom-center overlay: monospace, lists `<key>  label` pairs
- Attach dismiss keydown listener (same pattern as `hideHelp`)
- Store element in `whichKeyEl`

### `hideWhichKey()`

Clear `whichKeyTimer`, remove `whichKeyEl`, remove listener, null all three fields.

### Integration in `handleKey`

```typescript
// After buffer.push(key):
if (this.buffer.length === 1 && PREFIXES.has(key)) {
  this.showWhichKey(key)
} else {
  this.hideWhichKey()
}
```

```typescript
// In clearSequence():
this.hideWhichKey()
```

Also call `this.hideWhichKey()` in `destroyed()`.

**Acceptance:**
- `g` alone → overlay appears after 300ms with all g-group second keys
- `g s` quickly → overlay never appears (dismissed before 300ms fires)
- `Esc` in normal mode → clears sequence AND hides which-key overlay
- No listener leaks on `destroyed()`
